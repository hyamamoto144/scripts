# format_index.py

`extract_index.py` が出力したタブ区切りの索引ファイル（`index.txt` / `index_merged.txt`）を読み込み、
見出し付き・ソート済みの最終形の索引テキストに整形するスクリプトです。

```
extract_index.py（抽出）→ index.txt / index_merged.txt → format_index.py（整形）→ 最終版
```

これまでスプレッドシート上で手作業していた「グルーピング」「五十音・アルファベット順ソート」
「複数ファイルにまたがる同一語のページ番号統合」を自動化します。

## できること

- 語を以下の見出しに自動分類（フラットな1階層構成）

  ```
  ■■■索引
  ■■記号・数字（数字・記号始まり・分類不能な語をまとめて格納）
  ■■A
  ■■B
  ...
  ■■Z
  ■■あ
  ■■い
  ...
  ■■ん（46音、1文字ずつ）
  ```

- 濁音・半濁音・拗音・促音は、対応する清音の見出しに畳み込む
  （例:「が」→ あ見出しではなく**か**見出し、「ぎゃ」→ **き**見出し、「べ」→ **へ**見出し）
- 英字の見出しは大文字小文字を区別せず（`Amazon` も `agent.py` も同じA見出し内で
  ケース非依存にソート）
- 同じ単語が複数ファイル（複数章のPDF）にまたがって出現する場合、ページ番号を
  すべて集めて重複排除し、数値順に「、」区切りでまとめる
- 分類できない語（記号のみ、コード断片のハイライトなど）を削除せず
  「記号・数字」に残すため、抽出漏れに気づきやすい

## セットアップ（初回のみ）

`extract_index.py` と同じPython環境で動作します。追加の依存パッケージはありません
（標準ライブラリのみで動作します。読み仮名の変換自体は `extract_index.py` 側の
pykakasiで事前に済んでいる前提です）。

## 使い方

```bash
python format_index.py index_merged.txt
```

同じフォルダに `index_merged_formatted.txt` が生成されます。

出力先を指定したい場合:

```bash
python format_index.py index_merged.txt -o 最終索引.txt
```

`index.txt`（重複統合前の全件版）をそのまま渡しても、`format_index.py` 側で
同一語のページ番号をまとめ直すため問題なく使えます。

### オプション一覧

| オプション | 説明 | 既定値 |
|---|---|---|
| `input_path` | `extract_index.py` が出力したタブ区切りファイル | （必須） |
| `-o`, `--output` | 出力ファイルパス | 入力ファイル名 + `_formatted.txt` |

## 入力ファイルの形式

タブ区切りで、ヘッダー行に `単語` `読み` `ページ数`（`ファイル名` 列があっても無視されます）が
含まれている必要があります。`extract_index.py` の出力（`index.txt` / `index_merged.txt`）を
そのまま渡せます。

## 出力例

```
■■■索引
■■記号・数字（数字・記号始まり・分類不能な語をまとめて格納）
1_prompts_host.py	89
4_streamable_http_server.py	109

■■A
Agent Development Kit	10
AgentCore Gateway	11、62、241

■■あ
アウトバウンド認証	242

■■い
インバウンド認証	242、253
```

## うまく分類できないときは

- **「記号・数字」に大量の断片(コードの一部、記号だけの文字列など)が入る**
  これは`format_index.py`の分類ミスではなく、`extract_index.py`側でのハイライト誤検出
  （コードや記号を誤ってハイライトとして拾ってしまうケース）に由来します。
  索引に載せるべきでないものはこの中から手動で削除してください。

- **想定と違う見出しに入る**
  かな見出しへの分類は「読み」列（`extract_index.py`がpykakasiで生成したカタカナ）の
  先頭1文字を使います。読みが空、または英数字から始まる語は「記号・数字」に入ります。
  読みの精度自体に問題がある場合は、`extract_index.py`側の元データを見直してください。

- **同じ語なのに別々の見出しに分かれる／統合されない**
  完全に同じ文字列（表記ゆれなし）でないと同一語とみなされません。
  「AI エージェント」と「AIエージェント」のような表記ゆれは、統合前に
  どちらかに揃えておく必要があります。

## 依存関係

- 標準ライブラリのみ（`argparse`, `csv`, `re`, `pathlib`）
- 入力ファイルの生成には `extract_index.py`（PyMuPDF, pdfplumber, pandas, pykakasi）が必要です
