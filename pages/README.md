# 分量見積もりツール

Markdown原稿から、判型ごとのページ数を見積もるツールである。`book-length-estimator.html`をブラウザで開くだけで動作し、外部通信は一切行わない。

## できること

- Markdown原稿を段落・見出し・コードブロックに分解し、判型に依存しない文字数・行数として集計する
- 集計結果を、選択した判型の「1行字数」「1ページ行数」「段数」に基づいてページ数へ変換する
- 見出しレベルを境界として章ごとに区切り、章別の内訳（本文行数・見出し行数・コード行数・合計行数・ページ数・コード比率）と全体合計を出す
- 複数の判型を切り替えて、変換フェーズだけを差し替える形で比較する
- 原稿ファイルを複数選択、またはドラッグ＆ドロップで読み込み、連結して合計する
- 判型・書体プリセットをJSONとして書き出し・読み込みする
- 段組みと行密度の目安を示す簡易な版面模式図を表示する

## 使い方

1. `book-length-estimator.html`をブラウザで開く。
2. 原稿を貼り付けるか、`.md`/`.txt`ファイルを選択またはドラッグ＆ドロップする。複数ファイルは選択・ドロップした順に連結される。
3. 章の境界に使う見出しレベル（H1〜H5）を選ぶ。
4. 判型・書体プリセットを確認し、必要であれば追加・編集する。
5. 基準にする判型を1つ選び、比較したい判型があれば追加でチェックする。
6. 「見積もりを計算」を押す。判型ごとの合計ページ数と、基準判型の章別内訳が表示される。

## 判型・書体プリセットの構造（JSON）

設定は`typesetPresets`という配列に、判型・書体の組ごとに1つのオブジェクトとして持たせる。

```json
{
  "typesetPresets": [
    {
      "id": "a5-1c-std",
      "name": "A5判 1段 標準書体",
      "default": true,
      "compareByDefault": true,
      "format": {
        "charsPerLine": 40,
        "linesPerPage": 34,
        "columns": 1
      },
      "design": {
        "headingLines": { "1": 4, "2": 3, "3": 2, "4": 1, "5": 1 },
        "codeCoefficient": 0.75,
        "calibrated": true
      }
    }
  ]
}
```

各キーの意味を次に示す。

| キー | 型 | 意味 |
|---|---|---|
| `id` | 文字列 | プリセットの識別子。重複させない |
| `name` | 文字列 | 画面に表示する名称 |
| `default` | 真偽値（省略可） | `true`にすると「基準にする」の既定選択になる。複数のプリセットに付けた場合は配列内で先に見つかったものが優先される |
| `compareByDefault` | 真偽値（省略可） | `true`にすると「比較に含める」の既定チェックになる |
| `format.charsPerLine` | 数値 | 1行あたりの字数 |
| `format.linesPerPage` | 数値 | 1段あたりの1ページの行数 |
| `format.columns` | 数値 | 段数。ページあたりの行数は`linesPerPage × columns`として算出する |
| `design.headingLines` | オブジェクト | 見出しレベル（1〜5）ごとの固定行数 |
| `design.codeCoefficient` | 数値 | コード1行を本文換算で何行分として数えるかの係数 |
| `design.calibrated` | 真偽値 | 係数が過去の実測に基づくかどうか。`false`にすると画面に「未校正」の注記が出る |

新しい判型（B5変形判のような変則サイズなど）は、この構造に沿ってオブジェクトを1つ追記するだけで使えるようになる。既存のプリセットの数値を変える必要はない。

## 計算の考え方

Markdownの集計は判型に依存しない。段落・見出し・コードブロックへの分解と文字数・行数の数え上げは原稿ごとに一度だけ行い、判型を切り替えるたびにやり直す必要はない。ページ数への変換だけが判型に依存する部分であり、段落は`切り上げ(文字数 ÷ 1行字数)`で行数化し、見出しは`design.headingLines`の固定値をそのまま使い、コードブロックは`行数 × codeCoefficient`を切り上げて使う。

## プリセットの保存について

判型・書体プリセットの追加・編集・削除、およびJSON読み込みは、操作するたびにブラウザ内（localStorage）へ自動保存される。ファイルを保存したうえで同じブラウザで開き直せば、以前の編集内容がそのまま引き継がれる。

- 別のブラウザ・別のPCへ設定を移すときは、「JSONを書き出す」で書き出したファイルを、移った先で「JSONを読み込む」から読み込む。
- 自動保存された内容を白紙に戻したいときは、「既定のプリセットに戻す」を使う。
- Claudeのプレビュー画面内でファイルを開いて操作した場合は自動保存が効かないことがある。ダウンロードした`book-length-estimator.html`を手元に保存し、そのファイルを直接ブラウザで開いて使う。

## 前提と制限

- 版面模式図は、段組みの数と行数に応じた線の密度だけを示す簡易な図であり、判型の実寸や約物処理を反映した紙面イメージではない。
- 禁則処理や字詰めの微調整は行っていないため、算出したページ数は実際の組版と数ページから10ページ程度前後することがある。
- Vivliostyle/VFMテンプレートを使った実測ベースのページ数算出は、このツールの対象外である。
