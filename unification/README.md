# 用語統一チェッカー

WZ Editorの「用語統一」機能を再現した、ブラウザ単体で動くツールです。サーバや外部通信は使わず、`term-unification-checker.html` を開くだけで使えます。

## できること

- WZ Editor形式の用語統一辞書をそのまま読み込み、原稿(Markdown)への適用候補を検出する
- 検出したマッチを、ルールごとに文脈つきで一覧表示し、1件ずつ、または一括で適用/スキップを選べる
- `.#カテゴリ` 単位でルール群をまるごとON/OFFできる(例: 「である調→ですます調」変換だけ止める)
- コードブロック・インラインコード・HTMLタグ・リンクURL部分は自動的に対象外にする
- 複数の章ファイルをまとめて読み込み、ファイルごとに個別ダウンロードできる
- 辞書を `{pattern, flag, expected}` 形式のJSONとしてエクスポートできる(`wzeditor-word-rules-parser` と同じ構造)

## 使い方

1. **辞書を読み込む**: 「1. 辞書」のテキストエリアにWZ形式の辞書ファイルを貼り付けるか、ファイル選択でアップロードし、「辞書を解析」を押す
   - 解析結果としてルール数・未対応オプションの警告・カテゴリ一覧が表示される
   - カテゴリのチェックを外すと、そのカテゴリのルールは検出対象から外れる
2. **原稿を読み込む**: 「2. 原稿」でMarkdownファイルを複数選択する(章ごとに別ファイルでもまとめて読み込める)
3. **検出する**: 「検出」を押すと、有効なルールそれぞれについて原稿中のマッチ件数が集計される
4. **確認する**: 左のルール一覧(マッチ件数バッジ付き、件数の多い順)から1つ選ぶと、右側にそのルールの全マッチが文脈つきで表示される
   - 各マッチにチェックボックスがあり、デフォルトはチェック済み(適用予定)
   - 文脈がおかしい・誤検出だと思うものはチェックを外す
   - ルールの `<コメント>`(辞書内の注記)があれば、判断の参考として一緒に表示される
   - 「全部適用」「全部スキップ」でそのルールのマッチを一括操作できる
5. **反映する**: 「3. 反映してダウンロード」で「反映を実行」を押すと、チェックが入っているマッチだけが原稿に反映され、ファイルごとに反映件数が表示される
   - 反映があったファイルには「ダウンロード」ボタンが出る(`元のファイル名.fixed.md` として保存される)
   - 別々のルールが原稿の同じ範囲を書き換えようとした場合は、後から処理した側が自動的にスキップされ、「N件は範囲重複のためスキップ」と表示される

辞書は「JSONでダウンロード」ボタンで `{pattern, flag, expected}` 形式のJSONとしてエクスポートできます。このJSONはそのまま辞書欄に貼り付ければ再読み込みできます。

## 辞書ファイルの形式

WZ Editorの用語統一辞書と同じ、タブ区切りのテキスト形式にそのまま対応しています。

```
変更後単語	変更前単語	前置文字,後置文字,オプション
```

- `#` で始まる行はコメント
- `.#` で始まる行はカテゴリの区切り(以降の行がそのカテゴリに属する)
- 変更前単語は `|` 区切りで複数指定できる(例: `貴方|貴女`)
- 前置文字・後置文字は `[文字]`(その文字がある場合のみ適用)、`[^文字]`(その文字がない場合のみ適用)で指定する
  - 内部的には正規表現の先読み・後読み(`(?=...)` `(?<=...)`)に変換される
  - 行頭・行末など直前直後に文字が存在しない場合は、`[^...]` 条件は自動的に成立し、`[...]` 条件は自動的に不成立になる(正規表現の標準動作に準拠)
- オプション欄(3つめのカンマ区切りフィールド)は `|` で複数指定できる
  - `RE`: 変更前単語を正規表現として解釈する(キャプチャ参照 `\1` `\2` は、変更後単語側で使うと `$1` `$2` に変換される)
  - `TOHAN` / `TOZEN` / `SENTENCE`: 未対応。指定されている行があれば解析時に警告として表示するが、処理はスキップされる
- `<...>` はその行の注記コメントで、確認UIにルールの補足として表示される
- 変更後単語が空でTOHANが指定されている行(このファイルでは英数字の全角→半角変換の1行のみ)は特別扱いで、マッチした文字列そのものを半角変換した結果を置換に使う(固定の置換文字列を持たない動的なルールのため)

## JSON形式での読み込み

辞書テキストの先頭が `{` または `[` の場合、WZ形式ではなくJSONとして読み込まれます。想定する形式は次の通りです([wzeditor-word-rules-parser](https://github.com/azu/wzeditor-word-rules-parser)と同じ構造)。

```json
[
  { "pattern": "クッキー|\\bCOOKIE\\b", "expected": "Cookie" },
  { "pattern": "Web Socket", "expected": "WebSocket" }
]
```

- `pattern`: マッチする正規表現(文字列)
- `flag`: 正規表現フラグ(省略可)
- `expected`: 置換後の文字列(`$1` などのキャプチャ参照を含められる)

## 除外される範囲

原稿中の以下の部分は、用語統一の対象から自動的に除外されます。

- フェンスコードブロック(`` ``` `` や `~~~` で囲まれた範囲)
- インラインコード(`` `...` ``)
- HTMLタグ(`<...>`、属性値も含む)
- Markdownリンクの宛先部分(`](URL)` のURL側)

見出しや本文の日本語部分、リンクの表示テキストなどは通常通り対象になります。

## 既知の制限

- `SENTENCE`(段落全体検索)・`TOZEN`(全角化)オプションは未実装です。これらが指定された行は解析時に警告表示されますが、ルール自体は無視されます(この辞書ファイルには現時点で該当行はありません)。
- 前置文字・後置文字は「直前・直後の1文字」を条件にする実装です。複数文字にまたがる条件には対応していません。
- 反映処理は、検出時点のテキストに対する一括適用です。あるルールの適用結果を踏まえて別のルールを再検出する、という段階的な処理はしていません(範囲が重なった場合は片方をスキップするのみ)。
